# AI-Assisted Software Engineering Workflows: Apple Inventory App

This repository explores and compares two distinct paradigms of building software with Artificial Intelligence: **Vibe Coding** (immediate, loose conversational prompting) and **Spec-Driven Development (SDD)** (document-driven, structured engineering). 

The underlying project is a Python-based Command-Line Apple Inventory Application that tracks various apple stock counts.

## Repository Branch Structure

This repository is organized into three distinct branches to showcase the progression of the development workflows:
* **`main`**: The clean architectural baseline containing the core Python engine without any advanced AI feature implementations.
* **`vibe-coding`**: The codebase featuring a CSV export extension generated using rapid, conversational AI prompting.
* **`specs`**: The codebase featuring a secure, rule-compliant CSV export extension generated by anchoring the AI to a strict markdown specification file.

## Core Project Baseline (`main`)
The baseline application manages a dictionary dataset of apple varieties (e.g., Honeycrisp, Gala) and executes terminal lookups of active inventory logs.
